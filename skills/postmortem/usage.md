# Postmortem Analysis

Enforces a structured postmortem ritual when failures occur. Produces blameless analysis with root cause investigation (5 Whys), timeline reconstruction, and actionable systemic improvements.

## Usage Examples

### Run a postmortem

```
We had two consecutive test failures during deployment — run a structured postmortem.
```

### Analyze a production incident

```
The API was down for 30 minutes — let's do a postmortem to find the root cause and prevent recurrence.
```

### Post-rollback analysis

```
We had to rollback the last release — do a postmortem to identify what went wrong.
```
